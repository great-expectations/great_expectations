<p>An Inferred Asset Data Connector used to connect to an SQL database.</p>
<p>This Data Connector determines Data Asset names by introspecting the database schema.</p>
<dl class="field-list simple">
<dt class="field-odd">Parameters</dt>
<dd class="field-odd"><ul class="simple">
<li><p><strong>name</strong> – The name of the Data Connector.</p></li>
<li><p><strong>datasource_name</strong> – The name of this Data Connector’s Datasource.</p></li>
<li><p><strong>execution_engine</strong> – The Execution Engine object to used by this Data Connector to read the data.</p></li>
<li><p><strong>data_asset_name_prefix</strong> – A prefix to prepend to all names of Data Assets inferred by this Data Connector.</p></li>
<li><p><strong>data_asset_name_suffix</strong> – A suffix to append to all names of Data Asset inferred by this Data Connector.</p></li>
<li><p><strong>include_schema_name</strong> – If True the Data Asset name  will include the schema as a prefix.</p></li>
<li><p><strong>splitter_method</strong> – A method to use to split the target table into multiple Batches.</p></li>
<li><p><strong>splitter_kwargs</strong> – Keyword arguments to pass to the splitter method.</p></li>
<li><p><strong>sampling_method</strong> – A method to use to downsample within a target Batch.</p></li>
<li><p><strong>sampling_kwargs</strong> – Keyword arguments to pass to sampling method.</p></li>
<li><p><strong>excluded_tables</strong> – A list of tables to ignore when inferring Data Asset names.</p></li>
<li><p><strong>included_tables</strong> – A list of tables to include when inferring Data Asset names. When provided, only Data Assets
matching this list will be inferred.</p></li>
<li><p><strong>skip_inapplicable_tables</strong> – If True, tables that can’t be successfully queried using sampling and splitter methods
are excluded from inferred data_asset_names. If False, the class will throw an error during initialization
if any such tables are encountered.</p></li>
<li><p><strong>introspection_directives</strong> – Arguments passed to the introspection method to guide introspection. These may be,
<cite>schema_name</cite>, which filters to a specific schema,
<cite>ignore_information_schemas_and_system_tables</cite>, which defaults to True,
<cite>information_schemas</cite>, a list of schemas to consider as informational,
<cite>system_tables</cite>, a list of tables to consider system tables,
and <cite>include_views</cite> which defaults to True.</p></li>
<li><p><strong>batch_spec_passthrough</strong> – Dictionary with keys that will be added directly to the batch spec.</p></li>
<li><p><strong>id</strong> – The unique identifier for this Data Connector used when running in cloud mode.</p></li>
</ul>
</dd>
