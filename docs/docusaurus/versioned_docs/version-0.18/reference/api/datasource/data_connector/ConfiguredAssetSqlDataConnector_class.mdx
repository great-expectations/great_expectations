<p>A DataConnector that requires explicit listing of SQL assets you want to connect to.</p>
<p>Being a Configured Asset Data Connector, it requires an explicit list of each Data Asset it can
connect to. While this allows for fine-grained control over which Data Assets may be accessed,
it requires more setup.</p>
<dl class="field-list simple">
<dt class="field-odd">Parameters</dt>
<dd class="field-odd"><ul class="simple">
<li><p><strong>name</strong> (<em>str</em>) – The name of this DataConnector</p></li>
<li><p><strong>datasource_name</strong> (<em>str</em>) – The name of the Datasource that contains it</p></li>
<li><p><strong>execution_engine</strong> (<a class="reference internal" href="/docs/0.18/reference/api/execution_engine/ExecutionEngine_class" title="great_expectations.execution_engine.ExecutionEngine"><em>ExecutionEngine</em></a>) – An ExecutionEngine</p></li>
<li><p><strong>include_schema_name</strong> (<em>bool</em>) – Should the data_asset_name include the schema as a prefix?</p></li>
<li><p><strong>splitter_method</strong> (<em>str</em>) – A method to split the target table into multiple Batches</p></li>
<li><p><strong>splitter_kwargs</strong> (<em>dict</em>) – Keyword arguments to pass to splitter_method</p></li>
<li><p><strong>sorters</strong> (<em>list</em>) – List if you want to override the default sort for the data_references</p></li>
<li><p><strong>sampling_method</strong> (<em>str</em>) – A method to downsample within a target Batch</p></li>
<li><p><strong>sampling_kwargs</strong> (<em>dict</em>) – Keyword arguments to pass to sampling_method</p></li>
<li><p><strong>batch_spec_passthrough</strong> (<em>dict</em>) – dictionary with keys that will be added directly to batch_spec</p></li>
