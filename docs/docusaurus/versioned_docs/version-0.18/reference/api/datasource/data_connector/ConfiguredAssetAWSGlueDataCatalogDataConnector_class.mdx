<p>A Configured Asset Data Connector used to connect to data through an AWS Glue Data Catalog.</p>
<p>Being a Configured Asset Data Connector, it requires an explicit list of each Data Asset it can
connect to. While this allows for fine-grained control over which Data Assets may be accessed,
it requires more setup.</p>
<dl class="field-list simple">
<dt class="field-odd">Parameters</dt>
<dd class="field-odd"><ul class="simple">
<li><p><strong>name</strong> – The name of the Data Connector.</p></li>
<li><p><strong>datasource_name</strong> – The name of this Data Connector’s Datasource.</p></li>
<li><p><strong>execution_engine</strong> – The Execution Engine object to used by this Data Connector to read the data.</p></li>
<li><p><strong>catalog_id</strong> – The catalog ID from which to retrieve data. If none is provided, the AWS account
ID is used by default. Make sure you use the same catalog ID as configured in your spark session.</p></li>
<li><p><strong>partitions</strong> – A list of partition keys to be defined for all Data Assets. The partitions defined in Data Asset
config will override the partitions defined in the connector level.</p></li>
<li><p><strong>assets</strong> – A mapping of Data Asset names to their configuration.</p></li>
<li><p><strong>boto3_options</strong> – Options passed to the <cite>boto3</cite> library.</p></li>
<li><p><strong>batch_spec_passthrough</strong> – Dictionary with keys that will be added directly to the batch spec.</p></li>
<li><p><strong>id</strong> – The unique identifier for this Data Connector used when running in cloud mode.</p></li>
</ul>
</dd>
