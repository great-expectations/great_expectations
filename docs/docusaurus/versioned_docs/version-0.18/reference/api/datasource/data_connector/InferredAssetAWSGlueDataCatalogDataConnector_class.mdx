<p>An Inferred Asset Data Connector used to connect to data through an AWS Glue Data Catalog.</p>
<p>This Data Connector operates on AWS Glue Data Catalog and determines the Data Asset name
implicitly, by listing all databases, tables, and partitions from AWS Glue Data Catalog.</p>
<dl class="field-list simple">
<dt class="field-odd">Parameters</dt>
<dd class="field-odd"><ul class="simple">
<li><p><strong>name</strong> – The name of the Data Connector.</p></li>
<li><p><strong>datasource_name</strong> – The name of this Data Connector’s Datasource.</p></li>
<li><p><strong>execution_engine</strong> – The Execution Engine object to used by this Data Connector to read the data.</p></li>
<li><p><strong>catalog_id</strong> – The catalog ID from which to retrieve data. If none is provided, the AWS account
ID is used by default. Make sure you use the same catalog ID as configured in your spark session.</p></li>
<li><p><strong>data_asset_name_prefix</strong> – A prefix to prepend to all names of Data Assets inferred by this Data Connector.</p></li>
<li><p><strong>data_asset_name_suffix</strong> – A suffix to append to all names of Data Asset inferred by this Data Connector.</p></li>
<li><p><strong>excluded_tables</strong> – A list of tables, in the form <cite>([database].[table])</cite>, to ignore when inferring Data Asset
names.</p></li>
<li><p><strong>included_tables</strong> – A list of tables, in the form <cite>([database].[table])</cite>, to include when inferring Data Asset
names. When provided, only Data Assets matching this list will be inferred.</p></li>
<li><p><strong>glue_introspection_directives</strong> – Arguments passed to the introspection method. Currently, the only available
directive is
<cite>database</cite> which filters to assets only in this database.</p></li>
<li><p><strong>boto3_options</strong> – Options passed to the <cite>boto3</cite> library.</p></li>
<li><p><strong>batch_spec_passthrough</strong> – Dictionary with keys that will be added directly to the batch spec.</p></li>
<li><p><strong>id</strong> – The unique identifier for this Data Connector used when running in cloud mode.</p></li>
</ul>
</dd>
</dl>
