<p>A BatchRequest is the way to specify which data Great Expectations will validate.</p>
<p>A Batch Request is provided to a Datasource in order to create a Batch.</p>
<dl class="simple">
<dt>-Relevant Documentation Links
-</dt>
<dd><ul class="simple">
<li><p><a class="reference external" href="https://docs.greatexpectations.io/docs/0.18/reference/learn/terms/batch_request">Batch Request</a></p></li>
</ul>
</dd>
</dl>
<p>The <cite>data_connector_query</cite> parameter can include an index slice:</p>
<p><CodeBlock language="python">{`{
    "index": "-3:"
}`}</CodeBlock></p>
<p>or it can include a filter:</p>
<p><CodeBlock language="python">{`{
    "batch_filter_parameters": {"year": "2020"}
}`}</CodeBlock></p>
<dl class="field-list simple">
<dt class="field-odd">Parameters</dt>
<dd class="field-odd"><ul class="simple">
<li><p><strong>datasource_name</strong> – name of the Datasource used to connect to the data</p></li>
<li><p><strong>data_connector_name</strong> – name of the DataConnector used to connect to the data</p></li>
<li><p><strong>data_asset_name</strong> – name of the DataAsset used to connect to the data</p></li>
<li><p><strong>data_connector_query</strong> – a dictionary of query parameters the DataConnector
should use to filter the batches returned from a BatchRequest</p></li>
<li><p><strong>limit</strong> – if specified, the maximum number of <em>batches</em> to be returned
(limit does not affect the number of records in each batch)</p></li>
<li><p><strong>batch_spec_passthrough</strong> – a dictionary of additional parameters that
the ExecutionEngine will use to obtain a specific set of data</p></li>
</ul>
</dd>
