<p>A RuntimeBatchRequest creates a Batch for a RuntimeDataConnector.</p>
<p>Instead of serving as a description of what data Great Expectations should
fetch, a RuntimeBatchRequest serves as a wrapper for data that is passed in
at runtime (as an in-memory dataframe, file/S3 path, or SQL query), with
user-provided identifiers for uniquely identifying the data.</p>

<p>runtime_parameters will vary depending on the Datasource used with the data.</p>
<p>For a dataframe:</p>
<p><CodeBlock language="python">{`{"batch_data": df}`}</CodeBlock></p>
<p>For a path on a filesystem:</p>
<p><CodeBlock language="python">{`    {"path": "/path/to/data/file.csv"}`}</CodeBlock></p>
<dl class="field-list simple">
<dt class="field-odd">Parameters</dt>
<dd class="field-odd"><ul class="simple">
<li><p><strong>datasource_name</strong> – name of the Datasource used to connect to the data</p></li>
<li><p><strong>data_connector_name</strong> – name of the DataConnector used to connect to the data</p></li>
<li><p><strong>data_asset_name</strong> – name of the DataAsset used to connect to the data</p></li>
<li><p><strong>runtime_parameters</strong> – a dictionary containing the data to process,
a path to the data, or a query, depending on the associated Datasource</p></li>
<li><p><strong>batch_identifiers</strong> – a dictionary to serve as a persistent, unique
identifier for the data included in the Batch</p></li>
<li><p><strong>batch_spec_passthrough</strong> – a dictionary of additional parameters that
the ExecutionEngine will use to obtain a specific set of data</p></li>
</ul>
