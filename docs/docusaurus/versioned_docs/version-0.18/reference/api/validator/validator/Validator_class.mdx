<p>Run all expectations and return the outcome of the run.</p>
<dl class="field-list simple">
<dt class="field-odd">Parameters</dt>
<dd class="field-odd"><ul class="simple">
<li><p><strong>expectation_suite</strong> – If None, uses the Expectation Suite configuration generated during the current Validator session. If an <cite>ExpectationSuite</cite> object, uses it as the configuration. If a string, assumes it is a path to a JSON file, and loads it as the Expectation Sutie configuration.</p></li>
<li><p><strong>run_id</strong> – Used to identify this validation result as part of a collection of validations.</p></li>
<div class="deprecated">
<p><span class="versionmodified deprecated">Deprecated since version 0.13.0: </span>Only the str version of this argument is deprecated. run_id should be a RunIdentifier or dict. Support will be removed in 0.16.0.</p>
</div>
<li><p><strong>run_name</strong> – Used to identify this validation result as part of a collection of validations. Only used if a <cite>run_id</cite> is not passed. See DataContext for more information.</p></li>
<li><p><strong>run_time</strong> – Used to identify this validation result as part of a collection of validations. Only used if a <cite>run_id</cite> is not passed. See DataContext for more information.</p></li>
<li><p><strong>data_context</strong> – A datacontext object to use as part of validation for binding evaluation parameters and registering validation results. Overrides the Data Context configured when the Validator is instantiated.</p></li>
<li><p><strong>evaluation_parameters</strong> – If None, uses the evaluation_paramters from the Expectation Suite provided or as part of the Data Asset. If a dict, uses the evaluation parameters in the dictionary.</p></li>
<li><p><strong>catch_exceptions</strong> – If True, exceptions raised by tests will not end validation and will be described in the returned report.</p></li>
<li><p><strong>result_format</strong> – If None, uses the default value (‘BASIC’ or as specified). If string, the returned expectation output follows the specified format (‘BOOLEAN_ONLY’,’BASIC’, etc.).</p></li>
<li><p><strong>only_return_failures</strong> – If True, expectation results are only returned when <cite>success = False</cite>.</p></li>
<li><p><strong>checkpoint_name</strong> – Name of the Checkpoint which invoked this Validator.validate() call against an Expectation Suite. It will be added to <cite>meta</cite> field of the returned ExpectationSuiteValidationResult.</p></li>
</ul>
