<p>Sends a PagerDuty event.</p>
<p><CodeBlock language="yaml">{`- name: send_pagerduty_alert_on_validation_result
action:
  class_name: PagerdutyAlertAction
  api_key: \${pagerduty_api_key}
  routing_key: \${pagerduty_routing_key}
  notify_on: failure
  severity: critical`}</CodeBlock></p>
<dl class="field-list simple">
<dt class="field-odd">Parameters</dt>
<dd class="field-odd"><ul class="simple">
<li><p><strong>data_context</strong> – Data Context that is used by the Action.</p></li>
<li><p><strong>api_key</strong> – Events API v2 key for pagerduty.</p></li>
<li><p><strong>routing_key</strong> – The 32 character Integration Key for an integration on a service or on a global ruleset.</p></li>
<li><p><strong>notify_on</strong> – Specifies validation status that triggers notification. One of "all", "failure", "success".</p></li>
<li><p><strong>severity</strong> – The PagerDuty severity levels determine the level of urgency. One of "critical", "error", "warning", or "info".</p></li>

<p>Public entrypoint GX uses to trigger a ValidationAction.</p>
<p>When a ValidationAction is configured in a Checkpoint, this method gets called
after the Checkpoint produces an ExpectationSuiteValidationResult.</p>

<li><p><strong>data_asset</strong> – An instance of the Validator class.</p></li>
<li><p><strong>expectation_suite_identifier</strong> – Optionally, an instance of the ExpectationSuiteIdentifier class.</p></li>
<li><p><strong>checkpoint_identifier</strong> – Optionally, an Identifier for the Checkpoint.</p></li>
