<p>Sends an email to a given list of email addresses.</p>
<p><CodeBlock language="yaml">{`- name: send_email_on_validation_result
action:
  class_name: EmailAction
  notify_on: all # possible values: "all", "failure", "success"
  notify_with:
  renderer:
    # the class that implements the message to be sent
    # this is the default implementation, but you can
    # implement a custom one
    module_name: great_expectations.render.renderer.email_renderer
    class_name: EmailRenderer
  # put the actual following information in the uncommitted/config_variables.yml file
  # or pass in as environment variable
  smtp_address: \${smtp_address}
  smtp_port: \${smtp_port}
  sender_login: \${email_address}
  sender_password: \${sender_password}
  sender_alias: \${sender_alias} # useful to send an email as an alias
  receiver_emails: \${receiver_emails}
  use_tls: False
  use_ssl: True`}</CodeBlock></p>

<p>:param smtp_address: Address of the SMTP server used to send the email.
:param smtp_address: Port of the SMTP server used to send the email.
:param sender_login: Login used send the email.
:param sender_password: Password used to send the email.
:param sender_alias: Optional. Alias used to send the email (default = sender_login).
:param receiver_emails: Email addresses that will receive the email (separated by commas).
:param use_tls: Optional. Use of TLS to send the email (using either TLS or SSL is highly recommended).
:param use_ssl: Optional. Use of SSL to send the email (using either TLS or SSL is highly recommended).
:param notify_on: "Specifies validation status that triggers notification. One of "all", "failure", "success".
:param notify_with: Optional list of DataDocs site names to display  in Slack messages. Defaults to all.</p>
