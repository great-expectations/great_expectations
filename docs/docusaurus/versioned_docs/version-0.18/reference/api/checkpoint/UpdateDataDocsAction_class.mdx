<dd>
<p>Notify the site builders of all data docs sites of a Data Context that a validation result should be added to the data docs.</p>
<p>YAML configuration example:</p>
<p><CodeBlock language="yaml">{`- name: update_data_docs
action:
  class_name: UpdateDataDocsAction`}</CodeBlock></p>
<p>You can also instruct <code class="docutils literal notranslate"><span class="pre">UpdateDataDocsAction</span></code> to build only certain sites by providing a <code class="docutils literal notranslate"><span class="pre">site_names</span></code> key with a
list of sites to update:</p>
<p><CodeBlock language="yaml">{`- name: update_data_docs
action:
  class_name: UpdateDataDocsAction
  site_names:
    - local_site`}</CodeBlock></p>

<li><p><strong>site_names</strong> – Optional. A list of the names of sites to update.</p></li>

<p>Public entrypoint GX uses to trigger a ValidationAction.</p>
<p>When a ValidationAction is configured in a Checkpoint, this method gets called
after the Checkpoint produces an ExpectationSuiteValidationResult.</p>
