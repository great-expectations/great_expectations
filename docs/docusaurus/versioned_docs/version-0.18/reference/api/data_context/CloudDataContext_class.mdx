<p>Get the list of zero or more batches, based on a variety of flexible input types.</p>
<p><cite>get_batch_list</cite> is the main user-facing API for getting batches.
In contrast to virtually all other methods in the class, it does not require typed or nested inputs.
Instead, this method is intended to help the user pick the right parameters</p>
<p>This method attempts to return any number of batches, including an empty list.</p>
<dl class="field-list simple">
<dt class="field-odd">Parameters</dt>
<dd class="field-odd"><ul class="simple">
<li><p><strong>datasource_name</strong> – The name of the Datasource that defines the Data Asset to retrieve the batch for</p></li>
<li><p><strong>data_connector_name</strong> – The Data Connector within the datasource for the Data Asset</p></li>
<li><p><strong>data_asset_name</strong> – The name of the Data Asset within the Data Connector</p></li>
<li><p><strong>batch_request</strong> – Encapsulates all the parameters used here to retrieve a BatchList. Use either
<cite>batch_request</cite> or the other params (but not both)</p></li>
<li><p><strong>batch_data</strong> – Provides runtime data for the batch; is added as the key <cite>batch_data</cite> to
the <cite>runtime_parameters</cite> dictionary of a BatchRequest</p></li>
<li><p><strong>query</strong> – Provides runtime data for the batch; is added as the key <cite>query</cite> to
the <cite>runtime_parameters</cite> dictionary of a BatchRequest</p></li>
<li><p><strong>path</strong> – Provides runtime data for the batch; is added as the key <cite>path</cite> to
the <cite>runtime_parameters</cite> dictionary of a BatchRequest</p></li>
<li><p><strong>runtime_parameters</strong> – Specifies runtime parameters for the BatchRequest; can includes keys <cite>batch_data</cite>,
<cite>query</cite>, and <cite>path</cite></p></li>
<li><p><strong>data_connector_query</strong> – Used to specify connector query parameters; specifically <cite>batch_filter_parameters</cite>,
<cite>limit</cite>, <cite>index</cite>, and <cite>custom_filter_function</cite></p></li>
<li><p><strong>batch_identifiers</strong> – Any identifiers of batches for the BatchRequest</p></li>
<li><p><strong>batch_filter_parameters</strong> – Filter parameters used in the data connector query</p></li>
<li><p><strong>limit</strong> – Part of the data_connector_query, limits the number of batches in the batch list</p></li>
<li><p><strong>index</strong> – Part of the data_connector_query, used to specify the index of which batch to return. Negative
numbers retrieve from the end of the list (ex: <cite>-1</cite> retrieves the last or latest batch)</p></li>
<li><p><strong>custom_filter_function</strong> – A <cite>Callable</cite> function that accepts <cite>batch_identifiers</cite> and returns a <cite>bool</cite></p></li>
<li><p><strong>sampling_method</strong> – The method used to sample Batch data (see: Splitting and Sampling)</p></li>
<li><p><strong>sampling_kwargs</strong> – Arguments for the sampling method</p></li>
<li><p><strong>splitter_method</strong> – The method used to split the Data Asset into Batches</p></li>
<li><p><strong>splitter_kwargs</strong> – Arguments for the splitting method</p></li>
<li><p><strong>batch_spec_passthrough</strong> – Arguments specific to the <cite>ExecutionEngine</cite> that aid in Batch retrieval</p></li>
<li><p><strong>batch_request_options</strong> – Options for <cite>FluentBatchRequest</cite></p></li>
<li><p><strong>**kwargs</strong> – Used to specify either <cite>batch_identifiers</cite> or <cite>batch_filter_parameters</cite></p></li>

<p>Retrieve a Validator with a batch list and an <cite>ExpectationSuite</cite>.</p>
<p><cite>get_validator</cite> first calls <cite>get_batch_list</cite> to retrieve a batch list, then creates or retrieves
an <cite>ExpectationSuite</cite> used to validate the Batches in the list.</p>
<dl class="field-list simple">
<dt class="field-odd">Parameters</dt>
<dd class="field-odd"><ul class="simple">
<li><p><strong>datasource_name</strong> – The name of the Datasource that defines the Data Asset to retrieve the batch for</p></li>
<li><p><strong>data_connector_name</strong> – The Data Connector within the datasource for the Data Asset</p></li>
<li><p><strong>data_asset_name</strong> – The name of the Data Asset within the Data Connector</p></li>
<li><p><strong>batch</strong> – The Batch to use with the Validator</p></li>
<li><p><strong>batch_list</strong> – The List of Batches to use with the Validator</p></li>

<cite>batch_request</cite> or the other params (but not both)</p></li>

<li><p><strong>batch_data</strong> – Provides runtime data for the batch; is added as the key <cite>batch_data</cite> to
the <cite>runtime_parameters</cite> dictionary of a BatchRequest</p></li>
<li><p><strong>query</strong> – Provides runtime data for the batch; is added as the key <cite>query</cite> to
the <cite>runtime_parameters</cite> dictionary of a BatchRequest</p></li>
<li><p><strong>path</strong> – Provides runtime data for the batch; is added as the key <cite>path</cite> to
the <cite>runtime_parameters</cite> dictionary of a BatchRequest</p></li>
<li><p><strong>runtime_parameters</strong> – Specifies runtime parameters for the BatchRequest; can includes keys <cite>batch_data</cite>,
<cite>query</cite>, and <cite>path</cite></p></li>
<li><p><strong>data_connector_query</strong> – Used to specify connector query parameters; specifically <cite>batch_filter_parameters</cite>,
<cite>limit</cite>, <cite>index</cite>, and <cite>custom_filter_function</cite></p></li>
<li><p><strong>batch_identifiers</strong> – Any identifiers of batches for the BatchRequest</p></li>
<li><p><strong>batch_filter_parameters</strong> – Filter parameters used in the data connector query</p></li>
<li><p><strong>limit</strong> – Part of the data_connector_query, limits the number of batches in the batch list</p></li>
<li><p><strong>index</strong> – Part of the data_connector_query, used to specify the index of which batch to return. Negative
numbers retrieve from the end of the list (ex: <cite>-1</cite> retrieves the last or latest batch)</p></li>
<li><p><strong>custom_filter_function</strong> – A <cite>Callable</cite> function that accepts <cite>batch_identifiers</cite> and returns a <cite>bool</cite></p></li>
<li><p><strong>sampling_method</strong> – The method used to sample Batch data (see: Splitting and Sampling)</p></li>
<li><p><strong>sampling_kwargs</strong> – Arguments for the sampling method</p></li>
<li><p><strong>splitter_method</strong> – The method used to split the Data Asset into Batches</p></li>
<li><p><strong>splitter_kwargs</strong> – Arguments for the splitting method</p></li>
<li><p><strong>batch_spec_passthrough</strong> – Arguments specific to the <cite>ExecutionEngine</cite> that aid in Batch retrieval</p></li>

<li><p><strong>expectation_suite</strong> – The ExpectationSuite to use with the validator</p></li>
<li><p><strong>create_expectation_suite_with_name</strong> – Creates a Validator with a new ExpectationSuite with the provided name</p></li>
<li><p><strong>include_rendered_content</strong> – If <cite>True</cite> the ExpectationSuite will include rendered content when saved</p></li>
