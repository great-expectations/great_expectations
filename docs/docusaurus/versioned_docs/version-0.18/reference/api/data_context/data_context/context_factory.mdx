<p>Method to return the appropriate Data Context depending on parameters and environment.</p>
<dl>
<dt>Usage:</dt>
<dd>
<p><cite>import great_expectations as gx</cite></p>
<p><cite>my_context = gx.get_context(&lt;insert_your_parameters&gt;)</cite></p>
</dd>
</dl>
<p>This method returns the appropriate Data Context based on which parameters you’ve passed and / or your environment configuration:</p>
<ul class="simple">
<li><p>FileDataContext: Configuration stored in a file.</p></li>
<li><p>EphemeralDataContext: Configuration passed in at runtime.</p></li>
<li><p>CloudDataContext: Configuration stored in Great Expectations Cloud.</p></li>
</ul>
<p>Read on for more details about each of the Data Context types:</p>
<p><strong>FileDataContext:</strong> A Data Context configured via a yaml file. Returned by default if you have no cloud configuration set up and pass no parameters. If you pass context_root_dir, we will look for a great_expectations.yml configuration there. If not we will look at the following locations:</p>
<ul class="simple">
<li><p>Path defined in a GX_HOME environment variable.</p></li>
<li><p>The current directory.</p></li>
<li><p>Parent directories of the current directory (e.g. in case you invoke the CLI in a sub folder of your Great Expectations directory).</p></li>
</ul>
<p>Relevant parameters</p>
<ul class="simple">
<li><p>project_root_dir: Provide the project root where a GX project exists or will be scaffolded (contains the context root).</p></li>
<li><p>context_root_dir: Provide an alternative directory to look for GX config.</p></li>
<li><p>project_config: Optionally override the configuration on disk - only if <cite>context_root_dir</cite> is also provided.</p></li>
<li><p>runtime_environment: Optionally override specific configuration values.</p></li>
</ul>
<p><strong>EphemeralDataContext:</strong> A temporary, in-memory Data Context typically used in a pipeline. The default if you pass in only a project_config and have no cloud configuration set up.</p>
<p>Relevant parameters</p>
<ul class="simple">
<li><p>project_config: Used to configure the Data Context.</p></li>
<li><p>runtime_environment: Optionally override specific configuration values.</p></li>
</ul>
<p><strong>CloudDataContext:</strong> A Data Context whose configuration comes from Great Expectations Cloud. The default if you have a cloud configuration set up. Pass <cite>cloud_mode=False</cite> if you have a cloud configuration set up and you do not wish to create a CloudDataContext.</p>
<p>Cloud configuration can be set up by passing <cite>cloud_*</cite> parameters to <cite>get_context()</cite>, configuring cloud environment variables, or in a great_expectations.conf file.</p>
<p>Relevant parameters</p>
<ul class="simple">
<li><p>cloud_base_url: Override env var or great_expectations.conf file.</p></li>
<li><p>cloud_access_token: Override env var or great_expectations.conf file.</p></li>
<li><p>cloud_organization_id: Override env var or great_expectations.conf file.</p></li>
<li><p>cloud_mode: Set to True or False to explicitly enable/disable cloud mode.</p></li>
<li><p>project_config: Optionally override the cloud configuration.</p></li>
<li><p>runtime_environment: Optionally override specific configuration values.</p></li>
</ul>
<dl class="field-list simple">
<dt class="field-odd">Parameters</dt>
<dd class="field-odd"><ul class="simple">
<li><p><strong>project_config</strong> – In-memory configuration for Data Context.</p></li>
<li><p><strong>context_root_dir</strong> (<em>str</em><em> or </em><em>pathlib.Path</em>) – Path to directory that contains great_expectations.yml file</p></li>
<li><p><strong>project_root_dir</strong> (<em>str</em><em> or </em><em>pathlib.Path</em>) – Path to project root (contains context root with GX config)</p></li>
