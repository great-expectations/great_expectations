<p>Config class for DataContext.</p>
<p>The DataContextConfig holds all associated configuration parameters to build a Data Context. There are defaults set
for minimizing configuration in typical cases, but every parameter is configurable and all defaults are overridable.</p>
<p>In cases where the DataContext is instantitated without a yml file, the DataContextConfig can be instantiated directly.</p>
<dl class="field-list simple">
<dt class="field-odd">Parameters</dt>
<dd class="field-odd"><ul class="simple">
<li><p><strong>config_version</strong> (<em>Optional</em><em>[</em><em>float</em><em>]</em>) – config version of this DataContext.</p></li>
<li><p><strong>datasources</strong> (<em>Optional</em><em>[</em><em>Union</em><em>[</em><em>Dict</em><em>[</em><em>str</em><em>, </em><em>DatasourceConfig</em><em>]</em><em>, </em><em>Dict</em><em>[</em><em>str</em><em>, </em><em>Dict</em><em>[</em><em>str</em><em>, </em><em>Union</em><em>[</em><em>Dict</em><em>[</em><em>str</em><em>, </em><em>str</em><em>]</em><em>, </em><em>str</em><em>, </em><em>dict</em><em>]</em><em>]</em><em>]</em><em>]</em>) – DatasourceConfig or Dict containing configurations for Datasources associated with DataContext.</p></li>
<li><p><strong>fluent_datasources</strong> (<em>Optional</em><em>[</em><em>dict</em><em>]</em>) – temporary placeholder for Experimental Datasources.</p></li>
<li><p><strong>expectations_store_name</strong> (<em>Optional</em><em>[</em><em>str</em><em>]</em>) – name of ExpectationStore to be used by DataContext.</p></li>
<li><p><strong>validations_store_name</strong> (<em>Optional</em><em>[</em><em>str</em><em>]</em>) – name of ValidationsStore to be used by DataContext.</p></li>
<li><p><strong>evaluation_parameter_store_name</strong> (<em>Optional</em><em>[</em><em>str</em><em>]</em>) – name of EvaluationParamterStore to be used by DataContext.</p></li>
<li><p><strong>checkpoint_store_name</strong> (<em>Optional</em><em>[</em><em>str</em><em>]</em>) – name of CheckpointStore to be used by DataContext.</p></li>
<li><p><strong>profiler_store_name</strong> (<em>Optional</em><em>[</em><em>str</em><em>]</em>) – name of ProfilerStore to be used by DataContext.</p></li>
<li><p><strong>plugins_directory</strong> (<em>Optional</em><em>[</em><em>str</em><em>]</em>) – the directory in which custom plugin modules should be placed.</p></li>
<li><p><strong>validation_operators</strong> – list of validation operators configured by this DataContext.</p></li>
<div class="deprecated">
<p><span class="versionmodified deprecated">Deprecated since version 0.14.0.</span></p>
</div>
<li><p><strong>stores</strong> (<em>Optional</em><em>[</em><em>dict</em><em>]</em>) – single holder for all Stores associated with this DataContext.</p></li>
<li><p><strong>data_docs_sites</strong> (<em>Optional</em><em>[</em><em>dict</em><em>]</em>) – DataDocs sites associated with DataContext.</p></li>
<li><p><strong>notebooks</strong> (<em>Optional</em><em>[</em><em>NotebookConfig</em><em>]</em>) – Configurations for Jupyter Notebooks associated with DataContext, such as
the <cite>suite_edit</cite> Notebook.</p></li>
<li><p><strong>config_variables_file_path</strong> (<em>Optional</em><em>[</em><em>str</em><em>]</em>) – path for config_variables file, if used.</p></li>
<li><p><strong>anonymous_usage_statistics</strong> (<em>Optional</em><em>[</em><em>AnonymizedUsageStatisticsConfig</em><em>]</em>) – configuration for enabling or disabling
anonymous usage statistics for GX.</p></li>
<li><p><strong>store_backend_defaults</strong> (<em>Optional</em><em>[</em><em>BaseStoreBackendDefaults</em><em>]</em>) – define base defaults for platform specific StoreBackendDefaults.
For example, if you plan to store expectations, validations, and data_docs in s3 use the S3StoreBackendDefaults
and you may be able to specify fewer parameters.</p></li>
<li><p><strong>commented_map</strong> (<em>Optional</em><em>[</em><em>CommentedMap</em><em>]</em>) – the CommentedMap associated with DataContext configuration. Used when
instantiating with yml file.</p></li>
<li><p><strong>concurrency</strong> (<em>Optional</em><em>[</em><em>Union</em><em>[</em><em>ConcurrencyConfig</em><em>, </em><em>Dict</em><em>]</em><em>]</em>) – if enabled, Checkpoints associated with the DataContext
can run validations in parallel with multithreading.</p></li>
<li><p><strong>progress_bars</strong> (<em>Optional</em><em>[</em><em>ProgressBarsConfig</em><em>]</em>) – allows progress_bars to be enabled or disabled globally, for
profilers, or metrics calculations.</p></li>
<li><p><strong>include_rendered_content</strong> (<em>Optional</em><em>[</em><em>IncludedRenderedContentConfig</em><em>]</em>) – allows rendered content to be configured
globally, at the ExpectationSuite or ExpectationValidationResults-level.</p></li>
